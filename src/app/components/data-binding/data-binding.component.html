-------------------------- What is Data Binding --------------------------

data-binding works!

1. Interpolation (One-Way Binding: Component to View)

Description: Displays data from the component in the view (HTML) using two curly braces {{ '{{ }}' }}.

Use Case: Useful for binding string, number, or computed data directly in the template.


First Name: {{ firstName }}

RollNo: {{ rollNo }}

isActive: {{ isActive }}

Current Date: {{ currentDate }}


2. Property Binding (One-Way Binding: Component to View)

Description: Binds a property of an HTML element or a directive to a component property using [property] syntax.

Use Case: Useful for setting attributes like src, disabled, hidden, etc.




3. Event Binding (One-Way Binding: View to Component)

Description: Binds an event from the view to a method in the component using (event) syntax. Use Case:

Useful for handling user actions like clicks, keypresses, etc.





4. Two-Way Binding (Component to View and View to Component)

Description: Allows simultaneous data flow between the component and the view using [(ngModel)]. Requires FormsModule to be imported in AppModule.

Use Case: Useful for forms and inputs.
